# 中式九球计分系统功能说明

## 一、功能概述

中式九球计分系统是一款专为中式九球比赛设计的计分工具，支持2-5人同时比赛，提供完整的计分、统计、计时和操作记录功能。

## 二、页面结构

### 1. 比赛配置页 (`/pages/game/nineball/config`)
用于设置比赛参数，包括：
- 目标球数（4-9球）
- 玩家人数（2-5人）及昵称
- 基础分数
- 胜负分值（普胜、小金、大金、黄金九）
- 链（连续进球附加分）设置
- 倍数设置
- 比赛时间/局数

### 2. 记分板页 (`/pages/game/nineball/index`)
核心比赛页面，包含：
- 玩家卡片（垂直排列）
- 实时比分显示
- 统计数据（犯规、伴彩、普胜、大金、小金、黄金九）
- 操作按钮组
- 计时器
- 击球顺序悬浮框

### 3. 操作记录页 (`/pages/game/nineball/records`)
记录所有操作历史，支持清空功能。

## 三、核心规则

### 1. 选手顺序规则
- **初始顺序**: 按玩家ID升序排列（1→2→3→4→5）
- **赢局后**: 赢家第一位开球，上家（输家）第二位，其他玩家按原顺序排列
- **上家**: 当前选手的上一个击球者（犯规时的得分对手）
- **下家**: 当前选手的下一个击球者

### 2. 犯规规则
- 犯规者 -1分
- 上家（对手）+1分
- 下家获得自由球
- 重置犯规者连续进球数

### 3. 胜负计分规则

| 类型 | 默认分数 | 计分范围 | 说明 |
|------|---------|---------|------|
| 普胜 | 4分 | 上家 | 仅扣上家分数 |
| 小金 | 7分 | 可配置 | 默认上家，可设为全场 |
| 大金 | 10分 | 可配置 | 默认全场，可设为上家 |
| 黄金九 | 4分 | 可配置 | 默认全场，可设为上家 |

**计分逻辑**:
- 赢家：当前分数 + 胜负分值 × 倍数
- 输家：当前分数 - 胜负分值 × 倍数

### 4. 伴彩规则
- 自己 +伴彩分数（默认2分）
- 上家 -伴彩分数
- 仅当开启链功能时可用

### 5. 大金/黄金九禁用规则
- 点击换选手或犯规后，大金和黄金九按钮禁用
- 直到下一局比赛开始（点击任意胜负按钮）后重新启用

## 四、操作说明

### 按钮列表

| 按钮 | 图标 | 功能 | 说明 |
|------|------|------|------|
| 换选手 | 🔄 | 切换到下一位选手 | 禁用大金/黄金九 |
| 犯规 | ⛔ | 记录犯规 | 犯规者-1，上家+1 |
| 普胜 | 🏆 | 记录普胜 | 赢家+分，上家-分 |
| 小金 | 🥈 | 记录小金 | 可配置计分范围 |
| 大金 | 🥇 | 记录大金 | 可配置计分范围 |
| 黄金九 | 🌟 | 记录黄金九 | 可配置计分范围 |
| 伴彩 | 🎁 | 记录伴彩 | 需开启链功能 |

### 顶部操作

| 按钮 | 功能 |
|------|------|
| 返回 | 返回上一级页面 |
| 撤销 | 撤销上一步操作 |
| 记录 | 查看操作记录 |
| 重置 | 重新开始比赛 |

## 五、配置选项

### 得分设置

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| 基础分数 | 100 | 每位选手初始分数 |
| 普胜 | 4 | 普通胜利得分 |
| 小金 | 7 | 小金胜利得分 |
| 小金计分 | 上家 | 可选"上家"或"全场" |
| 大金 | 10 | 大金胜利得分 |
| 大金计分 | 全场 | 可选"上家"或"全场" |
| 黄金九 | 4 | 黄金九胜利得分 |
| 黄金九计分 | 全场 | 可选"上家"或"全场" |

### 链（连续进球附加分）

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| 启用链 | 关闭 | 是否启用伴彩功能 |
| 链分数 | 2 | 伴彩加分值 |

### 点（倍数）

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| 启用点 | 关闭 | 是否启用倍数 |
| 点倍数 | 2 | 倍数数值 |

### 比赛设置

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| 比赛时间 | 无 | 倒计时时间（分钟） |
| 比赛局数 | 无 | 目标局数 |

## 六、计时器功能

- **正计时模式**: 未配置比赛时间时，显示累计比赛时间
- **倒计时模式**: 配置比赛时间后，显示剩余时间
- **开始/停止**: 可随时开始或停止计时

## 七、数据持久化

所有数据均保存到本地存储：
- 比赛配置
- 玩家数据（分数、统计）
- 操作记录
- 游戏状态（当前选手、击球顺序）

## 八、技术栈

- **框架**: UniApp + Vue3
- **语言**: TypeScript
- **样式**: SCSS
- **存储**: uni.setStorageSync
