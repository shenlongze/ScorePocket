
# 台球比分记录小程序开发计划

## 一、需求分析

### 1.1 核心模块

| 模块 | 功能描述 | 状态 |
|------|----------|------|
| 抓迷糊模块 | 3~10人混战计分，随机球分配，淘汰判定，实时同步 | 核心需求 |
| 赛事图谱模块 | 单败/双败赛制可视化，抽签晋级，海报生成 | 核心需求 |
| 基础计分 | 5类玩法支持，手动计分，犯规处理 | 基础需求 |
| 联机系统 | 房间码/NFC配对，实时数据同步 | 基础需求 |
| 权限系统 | 免费/会员/单次付费分级管控 | 基础需求 |
| 广告系统 | 开屏、横幅、激励视频广告集成 | 附加需求 |

### 1.2 技术栈

- **框架**: UniApp (Vue 3)
- **构建工具**: Vite 5.x
- **图表组件**: echarts-for-uniapp
- **WebSocket**: Uni-push 2.0
- **样式**: SCSS


**淘汰判定逻辑**：
- 手动标记进球时校验是否为自身专属球
- 是专属球则标记淘汰，同步至所有设备
- 仅剩自身专属球时判定胜利
- 采用时间戳解决多设备冲突

### 3.2 赛事图谱模块

**赛制支持**：
- 单败淘汰赛：输一场淘汰
- 双败淘汰赛：输一场入败者组，再输淘汰

**可视化实现**：
- 使用 echarts-for-uniapp 绘制
- 支持手势缩放、全屏展示
- 实时更新晋级状态

### 3.3 联机系统

- **房间码配对**：6位数字房间码
- **NFC配对**：碰一碰触发，仅传递房间ID
- **数据同步**：WebSocket实时双向同步

### 3.4 权限系统

| 权限级别 | 功能限制 |
|----------|----------|
| 免费用户 | 基础1V1计分、查看文字版对阵 |
| 会员 | 全人数玩法、自定义球数、创建赛事 |
| 单次付费 | 临时解锁特定功能 |

## 六、风险评估

| 风险 | 描述 | 应对方案 |
|------|------|----------|
| WebSocket连接不稳定 | 网络波动导致同步延迟 | 心跳重连机制、离线缓存 |
| NFC兼容性 | 部分手机不支持NFC | 提供房间码备选方案 |
| 小程序审核 | 广告展示不合规 | 严格按照微信规范开发 |
| 性能问题 | 图谱渲染卡顿 | 虚拟滚动、按需渲染 |
