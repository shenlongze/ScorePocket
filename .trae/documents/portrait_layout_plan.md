# 竖屏布局排版优化计划

## 现状分析

当前竖屏布局 (`layout-portrait`) 的结构：
- 两个玩家信息卡片垂直排列
- 每个玩家卡片包含：头像、名称、分数、加减按钮、统计信息
- 中间用 VS 分隔

## 问题识别

1. **空间利用低效**：玩家卡片垂直堆叠，每个卡片高度较高
2. **布局层次不清晰**：头像、分数、操作按钮的排列不够直观
3. **视觉平衡不佳**：上下两个玩家卡片占用相似空间，中间 VS 区域显得单薄

## 优化方案

### 1. 调整玩家卡片布局

将竖版卡片改为更紧凑的水平布局：
- 头像 + 名称在左侧
- 分数居中显示（更大更醒目）
- 加减按钮在右侧
- 统计信息移至底部

### 2. 优化整体布局结构

```
┌─────────────────────────────────┐
│  玩家1  [头像] [名称]  [分数]  [+-]  │
│         [统计信息]                   │
├─────────────────────────────────┤
│              VS                   │
├─────────────────────────────────┤
│  玩家2  [头像] [名称]  [分数]  [+-]  │
│         [统计信息]                   │
└─────────────────────────────────┘
```

### 3. 样式优化

- 增加卡片内边距和圆角
- 使用渐变背景增强层次感
- 调整字体大小和颜色对比度

## 预期效果

1. 更紧凑的竖屏布局，充分利用横向空间
2. 分数显示更醒目，操作更便捷
3. 整体视觉层次更清晰

## 风险评估

- **低风险**：仅修改竖屏布局样式，不影响横屏布局
- **兼容性**：使用标准 flexbox 布局，兼容性良好
